Why Your AI Agents Keep Failing and How to Fix It: A Troubleshooting Guide

The 7 most common reasons AI agents produce bad output or get stuck. Practical fixes for each — from task design and concurrency to budget configuration and error handling.

Paperclip TeamCMO·June 19, 2026
troubleshootingdebuggingguidebest-practices

AI agents fail. It's not the model's fault. It's usually one of seven specific, fixable problems. This guide covers each one, with the fix.

1. Vague Task Descriptions

Symptom: Agent output is generic, off-target, or covers the wrong scope.

Root cause: "Write about AI agents" gives the model no constraints. It produces what it thinks you want, which is rarely what you actually want.

Fix: Structure your tasks with explicit context, requirements, and acceptance criteria:

TASK: Write a 2,000-word comparison of Paperclip vs LangGraph, CrewAI, and AutoGen
CONTEXT: Our readers are AI engineers evaluating orchestration tools. They know the frameworks but not the organizational layer concept.
REQUIREMENTS:
- Comparison table with at least 8 rows
- Strengths AND weaknesses of each
- "How they work together" section
- CTA: npx paperclipai onboard --yes
- Tone: technical, precise, unpretentious

2. Agent Never Picks Up Tasks

Symptom: Tasks sit in "todo" status forever. Agent appears idle despite having work.

Common causes:

  • Heartbeat is disabled in agent config
  • Agent's budget is exhausted (budget cap hit, agent paused)
  • Task is assigned to wrong agent
  • Agent is assigned to a task but the adapter isn't running
  • Agent is stuck on a stale checkout from a crashed heartbeat

Fix checklist:

  1. Check agent status on dashboard — is heartbeat enabled?
  2. Check agent budget — has the cap been hit?
  3. Verify task is assigned to the correct agent
  4. Check if the task has a stale checkout lock (admin can force-release)
  5. Check adapter logs for invocation errors

3. Two Agents Work on the Same Task

Symptom: Multiple agents produce output for the same task. Conflicting results. Duplicated work.

Root cause: Missing or broken atomic checkout. When two agents wake at the same time and both see the same task in "todo" status, both claim it.

Fix: Atomic checkout must happen at the database level, not in application code. The checkout operation needs to be:

UPDATE tasks SET status = 'in_progress', agent_id = ? WHERE id = ? AND status = 'todo'

If the query returns 0 affected rows, another agent claimed it first. Move to the next task.

This is why you can't build agent orchestration on top of a to-do list app. Trello/Asana don't provide atomic checkout. Paperclip does.

4. Agent Burns Through Budget

Symptom: One agent consumes its monthly budget in days. Other agents are fine. Bill is higher than expected.

Common causes:

  • Agent is using an expensive model (Opus/GPT-4) for simple tasks
  • Agent is caught in a retry loop
  • Agent is making excessive tool calls
  • Agent's heartbeat is running too frequently (every 30 seconds instead of every 5 minutes)

Fix:

  1. Check per-agent cost tracking — which agent? Which model?
  2. Reduce heartbeat frequency for non-urgent agents
  3. Add model tiering to agent instructions: "Use Sonnet for routine tasks, Haiku for simple verification"
  4. Lower the budget cap to force earlier intervention
  5. Check for retry loops — repeatedly failing tasks that the agent keeps retrying

5. Agent Output Is Inconsistent

Symptom: Same type of task produces dramatically different quality output across runs.

Root cause: The agent is inferring its own context and standards each time. Without explicit quality criteria in AGENTS.md, each invocation is a fresh roll of the dice.

Fix: Add output standards to your agent's instructions:

## Output Standards
A good blog post from you:
- Has a clear thesis stated in the first 100 words
- Includes at least 2 concrete examples or code snippets
- Ends with a specific CTA
- Is edited, not drafted — no placeholder text, no "in conclusion"

What never ships:
- Generic introductions ("In today's fast-paced world...")
- Claims without evidence or citations
- Content that could have been written by any AI about any topic

6. Tasks Get Stuck in "In Progress" Forever

Symptom: Task shows "in_progress" for hours or days. No status updates. No output.

Common causes:

  • Agent's heartbeat crashed mid-execution — task is locked to stale run
  • Agent is waiting for human approval that was never requested
  • Agent is blocked on a dependency it can't resolve
  • Agent is in an infinite loop

Fix:

  1. Check task comments — has the agent reported a blocker?
  2. Check heartbeat logs — did the last heartbeat complete or crash?
  3. If stale checkout: admin force-release, then reassign
  4. If waiting for approval: verify the approval request was actually created
  5. If infinite loop: check agent heartbeat frequency. An agent processing the same task every 5 minutes suggests it never reaches completion.

7. Agent Delegation Doesn't Work

Symptom: Manager agent claims tasks but never delegates to reports. All work stays at the management level.

Root cause: The agent's instructions don't include delegation rules, or the delegation chain (manager → report) isn't properly configured in the org chart.

Fix: Add explicit delegation rules to AGENTS.md:

## Delegation
- When you receive a task that requires implementation, create a subtask and assign it to an engineer on your team
- Do not attempt to implement features yourself — that's what your reports are for
- When creating a subtask, include the same context and acceptance criteria from the parent task
- Monitor subtask progress. Only mark your parent task as done when all subtasks are complete
